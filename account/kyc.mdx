KYC verifies a customer identity when required for security, compliance, or abuse prevention.

## Validation flow

```text
pending -> submitted -> under_review -> approved | rejected
```

## Submission statuses

| Status | Meaning |
|---|---|
| `pending` | No KYC file has been submitted. |
| `submitted` | Information has been sent. |
| `under_review` | The file is currently being checked. |
| `approved` | The file has been approved. |
| `rejected` | The file was rejected or needs correction. |

## Account statuses

| Status | Meaning |
|---|---|
| `unverified` | Account created, verification not completed. |
| `submitted` | Documents were sent. |
| `verified` | Account is verified. |
| `suspended` | Account is temporarily suspended. |
| `disabled` | Account is disabled. |
| `terminated` | Account was terminated or removed. |

## Tips to avoid rejection

- Use readable documents.
- Provide details that match the customer profile.
- Do not hide required information.
- Do not submit multiple conflicting files.

<Warning>
Do not send KYC documents through unofficial channels. Use only the official form or instructions.
</Warning>
