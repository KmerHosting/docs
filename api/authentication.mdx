The API has two access levels.

## Public endpoints

Public endpoints can be called without a session. Examples include:

- public content;
- catalog;
- registration;
- password reset request;
- newsletter subscription;
- trial requests.

## User endpoints

User endpoints require a valid session.

```text
authjs.session-token
```

This cookie is managed by the application when the user is signed in.

<Warning>
Do not copy a session cookie into a shared script. A session cookie grants access to the linked account.
</Warning>
