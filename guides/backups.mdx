Backups reduce the risk of data loss, but they still need to be tested.

## What to back up

- Website files.
- Database.
- Email data when needed.
- Configuration files.
- Private environment variables stored in a secure vault.

## Best practices

- Keep multiple versions.
- Store one copy outside the main server.
- Test the restoration process.
- Document the recovery steps.

<Warning>
Never store public SQL dumps or KYC backups in a public Git repository.
</Warning>
