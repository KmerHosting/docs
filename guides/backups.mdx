A backup is useful only when it is current, independent, restorable, and tested.

## Shared Hosting

Use available DirectAdmin backup tools for website files, databases, email, and account configuration. Download important backups outside the hosting account.

## Tiny VPS and VPS

Back up application files, environment configuration, databases, persistent container volumes, and reverse-proxy configuration. A provider snapshot is useful for recovery but should not be the only copy.

## Domains and DNS

Record nameservers and export DNS values before major changes.

## Professional Email

Use IMAP synchronization or mailbox export where appropriate. DNS records and mailbox passwords should be stored securely outside public documentation.

## Restore testing

Periodically restore to a separate test location. A backup that has never been restored is an unverified assumption.
