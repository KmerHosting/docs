KmerHosting supports two main deployment models:

- Shared hosting for standard websites, WordPress, and classic PHP apps.
- VPS for custom runtimes, root access, queues, workers, and containerized deployments.

## Choose the right platform

| Need | Shared hosting | VPS |
| --- | --- | --- |
| Static site or brochure website | Good fit | Works, but usually unnecessary |
| WordPress | Good fit | Better only when you need more control or scale |
| PHP plus MySQL application | Good fit | Good when you need custom packages or workers |
| Node.js, Python, Docker, Redis, workers | Not recommended | Good fit |
| Root access and firewall control | No | Yes |

## Deployment checklist

- Confirm whether the service is shared hosting or VPS before uploading anything.
- Point the domain or subdomain to the correct server or hosting account.
- Create the database and environment variables before the first app boot when the stack requires them.
- Enable SSL only after DNS resolves correctly.
- Validate backups, login, forms, and file uploads before announcing the site live.
