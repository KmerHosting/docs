title: "Maintenance"
description: "Prepare for planned KmerHosting maintenance and verify service state afterward."
icon: "wrench"
---


Maintenance is planned work that can affect a component for a defined period.

## Preparation

- Review the affected components and window.
- Avoid high-risk deployments during the maintenance period.
- Confirm backups before server or data changes.
- Notify application users when customer-facing downtime is possible.

## After the window

Check login, console access, service state, application health, DNS, email, billing, and monitoring. Report only persistent issues with exact timestamps.
