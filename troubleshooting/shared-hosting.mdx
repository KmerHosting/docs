## DirectAdmin login fails

Confirm the control-panel URL, username, and current hosting password. Do not use the customer account password unless the service explicitly says so.

## Website shows the wrong page

- Confirm the domain document root.
- Check `public_html` contents.
- Confirm DNS points to the correct hosting server.
- Remove default placeholder files only from the intended domain.

## PHP or database error

- Confirm the selected PHP version.
- Review error logs.
- Verify database host, name, user, and grants.
- Keep secrets out of public files and screenshots.

## Email or SSL error

Use the dedicated [email guide](/guides/email-configuration), [DNS guide](/guides/dns-management), and [SSL guide](/guides/ssl-certificates).
