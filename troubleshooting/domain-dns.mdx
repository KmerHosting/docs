Domain problems often involve several independent layers: registration, nameservers, DNS records, hosting attachment, application configuration, email records, and SSL.

## Diagnostic order

1. Confirm the domain is active and not expired.
2. Verify authoritative nameservers.
3. Query the required DNS record.
4. Confirm the target server or service is reachable.
5. Check the hosting domain attachment.
6. Test HTTP and HTTPS.
7. Check MX and TXT records separately for email.

## Propagation

Cached resolvers can show old values after a correct change. Compare multiple resolvers and wait for the previous TTL where necessary.
